# Definición de hecho común

## Propósito
Este documento recoge las condiciones comunes que deben cumplirse para dar por terminada la implementación de cualquier ejercicio del proyecto.

## Condiciones comunes
- La feature queda implementada de forma coherente en todos los proyectos afectados por el cambio (front-end y back-end).
- Existe al menos una prueba automatizada del flujo feliz.
- Existe al menos una prueba automatizada de validación o error significativo.
- Si durante la implementación cambia un contrato público, se actualiza la documentación de contratos vigente en el proyecto afectado.
- Las pruebas automatizadas relevantes del cambio quedan en estado satisfactorio.
